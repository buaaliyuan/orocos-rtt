Detailed Description

Hardware description interface.

The Hardware Description Interface provides a method to gather information about the CAN hardware and the functionality of the driver. All items in the structure CpStruct_HDI are constant and must be supplied by the designer of the CAN driver. The hardware description structure is available for every physical CAN channel.

Field Documentation

_U08 CpStruct_HDI::v_ControllerType
 

Constant value that identifies the used CAN controller chip. Possible values for this member are listed in the header file cpconst.h

char * CpStruct_HDI::v_DriverName
 

A null terminated string which holds the name of the specific CAN driver. The string can be used to distinguish between different vendors.

_U08 CpStruct_HDI::v_IRQNumber
 

Defines the number of the interrupt which is used. If the flag IRQHandler is set to 0, the value of IRQNumber will be undefined.

_U08 CpStruct_HDI::v_SupportFlags
 

Bit coded value that decribes the features of the CAN driver.

  • Bit 0/1: 0 = Standard Frame, 1 = Extended Frame passive, 2 = Extended Frame active
  • Bit 2: 0 = BasicCAN, 1 = FullCAN
  • Bit 3: 0 = No IRQ Handler, 1 = Has IRQ Handler
  • Bit 4: 0 = No identifier filter, 1 = software identifier filter
  • Bit 5: 0 = No timestamp, 1 = has timestamp
  • Bit 6: 0 = No user data field, 1 = has user data field
  • Bit 7: reserved

_U08 CpStruct_HDI::v_VersionMajor
 

Holds the major version number of the CANpie driver release (current value = 0).

_U08 CpStruct_HDI::v_VersionMinor
 

Holds the minor version number of the CANpie driver release (current value = 4).

char * CpStruct_HDI::v_VersionNumber
 

A null terminated string which holds the version of the specific CAN driver (e.g. DLL version, firmware version).
