Detailed Description

CANpie user functions.

The user functions always call the core functions, they never access the hardware directly. That means the user functions do not have to be modified when implementing the CANpie on an existing hardware. A typical user function is the writing of a CAN message to the FIFO buffer (e.g. CpUserMsgWrite() ).

The CAN driver is initialized with the function CpUserAppInit(). This routine will call several core functions for the correct setup of the CAN controller. The core functions called are:

  • CpFifoSetup()
  • CpCoreInitDriver()
  • CpUserBaudrate()
  • CpUserFilterAll()
The initial baudrate is set to 20kBit/s by default. The function CpUserAppInit() must be called before any other CANpie function.

Cp_EXPORT _U08 Cp_PREFIX CpUserBaudrate ( _U08 channel,
_U08 baud )
 

Setup baudrate.

Parameters:
channel   CAN channel of the hardware
baudrate   constant for the baudrate, taken from CP_BAUD enumeration
Returns:
Error code taken from the CpErr enumeration. If no error occured, the function will return CpErr_OK.

This function sets the baudrate of the selected CAN channel. Possible values for the baudrate are:
  • CP_BAUD_10K - 10 kBit/s
  • CP_BAUD_20K - 20 kBit/s
  • CP_BAUD_50K - 50 kBit/s
  • CP_BAUD_100K - 100 kBit/s
  • CP_BAUD_125K - 125 kBit/s
  • CP_BAUD_250K - 250 kBit/s
  • CP_BAUD_500K - 500 kBit/s
  • CP_BAUD_800K - 800 kBit/s
  • CP_BAUD_1M - 1 MBit/s
It simply calls the CpCoreBaudrate() function with bit timing values that are defined in the table "CpStruct_BitTiming". The values of this table depend on the used CAN controller and the controller clock frequency.

Cp_EXPORT _U08 Cp_PREFIX CpUserFilterAll ( _U08 channel,
_BIT enable )
 

Set global filter.

Parameters:
channel   CAN channel of the hardware
enable   TRUE for accepting all identifiers, FALSE for rejecting all identifiers

Returns:
Error code taken from the CpErr enumeration. If no error occured, the function will return CpErr_OK.

This function enables or disables the reception of CAN messages. If the flag enable is set to TRUE, all incoming CAN messages are accepted. If the flag enable is set to FALSE, all CAN messages are rejected. Please notice that this function only works for standard messages (2.0A, 11 bit identifier). This function is only available, if the member SupportSWFilter of the HDI structure (CpStruct_HDI) is set to TRUE.

Set filter for single message.

channel   CAN channel of the hardware
id   value of the identifier
enable   TRUE for accepting an identifier, FALSE for rejecting an identifier

Returns:
Error code taken from the CpErr enumeration. If no error occured, the function will return CpErr_OK.

This function sets a filter for CAN messages. If the flag enable is set to TRUE, the CAN identifier id is accepted. If the flag enable is set to FALSE, the reception of messages with the identifier id is not possible. Please notice that this function only works for standard messages (2.0A, 11 bit identifier). This function is only available, if the member SupportSWFilter of the HDI structure (CpStruct_HDI) is set to TRUE.
